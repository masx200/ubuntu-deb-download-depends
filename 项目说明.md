# 项目说明

## 📁 项目文件结构

```
ubuntu-deb-download-depends/
├── README.md                 # 完整使用文档
├── QUICKSTART.md            # 快速开始指南
├── 项目说明.md              # 本文件
├── download-packages.sh     # Linux/Mac版主脚本(完整版,支持清华源)
├── download-simple.sh       # 简化版Linux脚本(支持清华源)
├── download-packages.bat    # Windows批处理脚本
├── download-packages.ps1    # PowerShell脚本
└── download-remote.sh       # 远程服务器执行脚本(自动配置清华源)
```

## 🎯 项目目标

在 **远程服务器** (`**************`) 的 **Docker容器**
中运行Ubuntu,**使用清华大学镜像源**下载以下软件包及其**所有依赖**:

- nginx
- postgresql-12
- openjdk-11-jdk

然后将下载的 `.deb` 包用于**离线部署**到没有网络的目标机器。

## ✨ 清华大学镜像源特性

### 自动配置

脚本会自动:

1. 检测Ubuntu版本代号 (如: noble, noble, focal等)
2. 备份原始源配置文件
3. 根据系统版本选择正确的配置格式:
   - **DEB822格式**: Ubuntu 24.04+
   - **传统格式**: Ubuntu 24.04之前
4. 替换为 `mirrors.tuna.tsinghua.edu.cn`
5. 优先使用清华PostgreSQL镜像

### 优势

- 🚀 **下载速度提升**: 国内用户从清华源下载比官方源快10-50倍
- 🔄 **自动适配**: 无需手动配置,脚本自动识别系统版本
- 🔒 **安全可靠**: 保留GPG签名验证,确保软件包完整性
- 📝 **配置备份**: 自动备份原始配置,可随时恢复

### 支持的架构

- ✅ amd64 (x86_64) - 主要支持
- ⚠️ 如需其他架构(arm64, armhf等),请使用 `ubuntu-ports` 镜像

## 🚀 快速开始

### Windows用户 (最简单)

1. **双击运行**: `download-packages.bat`
2. 等待下载完成 (10-30分钟)
3. 从远程服务器复制文件:
   ```bash
   scp -r root@**************:/tmp/offline-packages ./ubuntu-offline-packages
   ```

```bash
chmod +x download-packages.sh

```

## 📦 下载内容

脚本会下载:

- **主软件包**: nginx, postgresql-12, openjdk-11-jdk
- **所有依赖**: 通过apt-rdepends递归分析获取
- **预计数量**: 300-500个deb包
- **预计大小**: 500MB - 1GB

## 🔧 技术特点

### ✅ 优势

- **环境隔离**: 在Docker容器中操作,不影响远程服务器
- **完整依赖**: 使用apt-rdepends确保下载所有依赖
- **可重复**: 每次运行都清理旧环境,保证干净状态
- **自动化**: 一键执行,无需手动干预

### ⚙️ 技术要求

- SSH客户端 (Windows 10+已内置)
- 远程服务器需安装Docker
- 远程服务器需能访问互联网
- 目标系统与下载系统版本一致 (如都是Ubuntu 24.04)

## ❓ 常见问题

### Q1: 为什么要在Docker中下载?

**A**:

- 保持远程服务器环境干净
- 确保依赖分析的准确性
- 可以随时重建环境

### Q2: 下载失败怎么办?

**A**:

- 检查网络连接
- 确认Docker正常运行
- 重新运行脚本(会自动清理)

### Q3: 如何验证下载完整性?

**A**:

```bash
# 查看包数量
ssh root@************** "ls -1 /tmp/offline-packages/*.deb | wc -l"

# 应该 > 300个
```

### Q4: 可以在不同Ubuntu版本间使用吗?

**A**:

- **不建议**。下载环境和目标环境的Ubuntu版本应一致
- 例如: 都在Ubuntu 24.04上下载,部署到Ubuntu 24.04

### Q5: 离线安装时遇到依赖冲突?

**A**:

```bash
sudo apt-get install -f -y  # 自动解决依赖
```

## ⚠️ 注意事项

1. **系统版本一致性**: 下载和部署的Ubuntu版本必须一致
2. **架构一致性**: amd64对amd64, arm64对arm64
3. **磁盘空间**: 确保远程服务器有足够空间 (>2GB)
4. **网络稳定**: 下载过程需要稳定的网络连接
5. **耐心等待**: 大量小文件下载可能需要较长时间
